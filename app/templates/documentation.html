General

The adjacency matrix format is a way of describing the relations in a network. It is an n√ón matrix, where n is the number of nodes.
The matrix is indexed by the unique node identifier in the columns as well as the rows.
The values in the matrix represent the weight of the relations. A weight of zero means that there is no relation.

The following table depicts the adjacency matrix for a network of 3 nodes, where Node1 is related to Node2 and Node3 (and vice versa).

Node1 Node2 Node3
Node1 0 1 1
Node2 1 0 0
Node3 1 0 0

If you want to know how to follow this format in the various supported file types, navigate to its documentation page on the left.

CSV/TXT

In CSV and TXT format, it is easy to create a file for an adjacency matrix. You can choose any separator you like.
To get a correct result, the first line must start with the separator, followed by the node identifiers separated by the separator.
Every line after that should begin with a node identifier, followed by the weights of the relations separated by the separator.

The adjacency matrix from the general page looks like this in CSV form (using default separator):

,Node1,Node2,Node3
Node1,0,1,1
Node2,1,0,0
Node3,1,0,0

Excel sheets

To get Excel sheets to work with our tool, you just need to make sure that it looks like the table on the general page in your Excel sheet editor.
Supported Excel sheet formats are: xls, xlsx, xlsm.

JSON

There are multiple ways to save a table in JSON. To get our tool to work correctly, your JSON file must follow a specific layout.
Here is a JSON string of network on the general page:

{
"Node1": {
"Node1":0,
"Node2":1,
"Node3":1,
},
"Node2": {
"Node1":1,
"Node2":0,
"Node3":0,
},
"Node3": {
"Node1":1,
"Node2":0,
"Node3":0,
},
}

 As you can see, this is a series of nested JSONs, with the node names as the key for the JSON describing the outgoing relations of that node.
